Web性能分析与评估

Google PageSpeed Insights

该工具是Google的评估工具，它会分析网站，并就如何改进性能和体验给出建议，一共会执行两次，一次是使用移动用户代理，一次是使用桌面用户代理，它分析性能分两个因素，折线以上为内容加载所需实现，以及整个页面加载所需时间

该工具无法检测本地URL，仅支持上线的Web网站。

Google Analytics批量报告( 无法访问，you know )

这个报告工具提供了网站访问者的数据，例如其位置、如何到达你的网站、在哪里花费了多少时间，以及其他统计信息

如果你的网站上已经有Google Analytics，那么你要做的就是登录并跟进。如果你尚未在网站上安装它，请使用Google账户登录http://www.google.com/analytics，并按照说明进行操作。这个过程只需花费很少的时间，并且需要将一小段JavaScript代码粘贴到站点的HTML中。之后需要等一到两天，让GoogleAnalytics收集数据。

请注意，向网站添加Google Analytics会带来法律问题。安装跟踪代码时，你要接受法律协议的条款。如果你是网站的唯一拥有者，那么可以自行决定，否则一定要得到网站拥有者的同意。如果你是一家大公司的开发人员，这一点很重要，因为法律审查在大公司是常见的流程。

查看计时信息

首字节时间（TTFB）：即从用户请求网页到响应的第一个字节到达之间的时间，与加载时间不同，加载时间是资源完全下载完成所需时间。

网络条件造成的TTFB较长的主要因素有：物理距离，服务器性能或者后台问题等，要了解一个请求所花费的时间，可在Chrome -> Network -> Waterfall中查看；或者单击一个条目在右侧选择Timing。

渲染性能检查工具

性能的另一个方面是页面的渲染速度，页面的初始渲染很重要，但是渲染后与网页的交互是否流畅也不容忽视。

理解浏览器如何渲染网页

步骤如下：

  1. 解析HTML创建DOM
  2. 解析CSS创建CSSOM
  3. DOM与CSSOM组合执行布局过程，创建UI
  4. 布局后将应用css和媒体内容进行绘制，结束时输出转换为像素显示在屏幕上
Google Chrome Performance

如何使用：

首先记录页面加载时发生的事情，先开始录制，然后刷新页面，在页面加载完成后结束录制。

这个工具捕获4种特定类型时间：

  1. 加载中 - 蓝色，网络相关事件以及HTML、CSS解析、图像解码等活动
  2. 执行脚本 - 黄色，与JavaScript相关，特定于DOM、垃圾收集等
  3. 渲染 - 紫色，与页面渲染相关所有事件，将CSS应用于HTML等活动，以及会导致重新渲染的活动，比如Js触发的HTML更改
  4. 绘制 - 绿色，将布局绘制到屏幕上的相关的事件，比如层合成和光栅化

大部分时间都耗在了不能识别的部分活动上，下面要找到他的问题：

Jank*

确定性能问题之前，必须定义页面性能的主要目标，很简单，目标就是最小化浏览器加载和渲染页面的时间，要做到这一点必须要击败Jank。

Jank是指交互和动画效果卡顿，或未能顺利渲染。如果使用的编程技术欠佳，那么即使是从网络快速加载的页面也会受到它的影响。

什么会引起Jank？

当单一帧中占用了太多CPU时间，就会发生这种情况

帧是浏览器在美妙显示时间内所做的工作量。

工作是指时间，比如加载、脚本执行、绘制、渲染等。

当帧堆积起来的时候，帧速率就会下降，可能是由于脚本触发太过频繁，加载事件花费的时间过长，以及其他导致渲染和绘制操作效率低下的活动造成的。

要将60帧率设定为目标，这是典型显示的最佳帧率。

大多数开发者工具都会测量帧速率，Performance面板以毫秒为计算单位，由于一秒60帧，简单计算出每帧有16.66毫秒的预算，减去浏览器在每帧中的开销，所以Google建议每帧有10ms的预算。

服务器压缩 - compression模块

服务器压缩的工作方式是用户从服务器请求网页，用户的请求附带Accept-Encoding头，向服务器告知浏览器可以使用的压缩格式，如果服务器能够按照Accept-Encoding头中指示的内容进行编码，它将用一个描述压缩方式和压缩内容的Content-Encoding头部信息进行回复。

因为网站下载的大部分内容往往是文本，可压缩性很好，几乎所有通用的浏览器都支持一种名为gzip的压缩方式，它在减小文本资源体积方面非常有效。

函数的执行上下文和作用域链

2021-01-27
前端

概念解释：

概念 意义
执行上下文[execution context] 描述JavaScript代码执行的抽象概念，所有JavaScript代码都是在某个执行上下文中运行的
激活对象[activeation] 进入新的执行上下文时创建出来，与新的执行上下文关联
作用域链[scope chain] 由对象列表或者对象链组成，构成了函数调用上下文环境的作用域
arguments 内部存储函数参数的类数组
内部属性[scope] 由对象列表或对象链组成，构成了函数调用上下环境的作用域
词法作用域 函数定义时根据函数代码嵌套关系而确定的作用域
动态作用域 函数执行时根据调用随时调整的作用域
