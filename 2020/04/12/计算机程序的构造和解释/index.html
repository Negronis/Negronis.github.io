1.1 程序设计的基本元素

每一种强有力的语言都提供了三种机制：
1. 基本表达形式，用于表示语言所关心的最简单个体
2. 组合的方法，通过他们可以从较简单的东西出发，构造复合元素
3. 抽象的方法，可以为复合对象命名，并将他们当做单元去操作

1.1.1 表达式
1. 基本式子: 486
2. 组合式：(4 + 2)
3. 前缀表示：(* 25 4 12)
4. 组合式嵌套：(+(*3 5)(-10 6))

对于复杂表达式，解释器也总是按照同样的基本循环来运作。

从终端读入一个表达式 - 对这个表达式求值 - 打印出得到的结果。

1.1.2 命名和环境
  1. 变量：提供一种通过名字去使用计算对象的方式，值是他对应的对象。
  2. define：Scheme中的定义方式：

    (defined size 2) //将2与size相关联。

  3. 他是Scheme中最简单的抽象方式，允许用一个简单名字去引用一个复杂结果。
  4. 一个Lisp程序通常总是由一大批相对简单的过程构成的
  5. 解析器需要维护某种存储能力，以便保持有关的名字-值对偶的归结，这种存储被称为** $\color{red}{环境}$ (更准确为全局环境，一个计算过程中有可能设计多个不同环境) **。
1.1.3 组合式的求值

把过程性思维有关的各种问题隔离出来

  1. 解析器按照下面过程工作：
    1. 求值该组合式的各个字表达式。
    2. 将作为最左子表达式(运算符)的值的过程应用于相应的实际参数(运算对象)。
  2. 为了实现对一个组合式的求职过程，必须先对每个元素执行相同求值过程，性质上这一求职过程是递归的 – 在自己的工作步骤中包含着调用这个规则本身的需要。

例如：对下面的表达式进行求值：

( * ( + 2 ( * 4 6 ) )
( + 3 5 7 ) )

可分解为如下树形结构：

  1. 需要将求值规则应用于4个不同的组合式，每个组合式用一个带分支的结点表示，有它发出的分支对应于组合式里的运算符和各个运算对象。
  2. 终端结点(不再发出分支的结点)表示的是运算符或者数值。
  3. 以树的观点观看这种求值过程，可以设想那些运算对象的值向上穿行，从终端结点开始，然后在越来越高的层次中组合起来。
  4. 树形累计：“值向上穿行”形式的求值形式。
  5. 在这里我们遇到的不是组合式而是基本表达式，处理这些基本情况的方式如下规定：
    a. 数的值就是它们所表示的值。
    b. 内部运算符的值就是能完成相应操作的机器指令序列。
    c. 其他名字的值就是在环境中关联于这一名字的那个对象。
  6. +和*一类的运算符也包含在全局环境中，并将对应的指令序列作为与之关联的值。
  7. 环境是具有普遍性的概念，为求值过程提供了一种上下文。
  8. 特殊形式：求职规则没有处理的特殊形式，比如define这种关联，表明了(define x 3)并不是一种组合式，每个特殊形式都有自身的求职规则，各种不同种类的表达式组成了程序设计语言的语法形式。
  9. 对各种表达式的求值规则可以描述为：简单的通用规则+处理特殊形式的专门规则$\color{red}{(语法糖 - 为可以采用统一形式描述的东西给出的另一种表面结构)}$
1.1.4 复合过程

  1. 任何一种强有力的设计语言应该包括：
    a. 数和算数运算符是基本的数据和过程。
    b. 组合式的嵌套提供了一种组织起多个操作的方法。
    c. 定义是一种受限的抽象手段，为名字关联相应的值。

  2. 过程定义：强大的抽象技术，可以为复合操作提供名字，然后可以将这种操作作为单元使用。

    参考描述平方的想法：

    (define (square x) (* x x ))
    //如果在JavaScript中
    function square(x){
    return x*x ;
    }

    可以描述为：

    这样就有了一个复合过程square，它表示将一个东西乘以它自身的操作。被乘的东西也给定了一个局部名称x。求值这个定义的结果就是创建起了一个复合过程，并关联与square。
    定义过程的一般形式：

    (define (<name><format parameters>)<body>)

    a. 是一个符号，过程定义将在环境中关联它。
    b. 形参，是一个名字，用在过程体中，用于表示过程应用的时候与它们对应的每个实际参数。
    c.是一个表达式，应用这一过程的时候，这个表达式中的形参将用与之对应的实参所取代，对取代后的表达式进行求职，产出过程应用的值。
    可以用square作为基本构件去定义其他过程，例如x²+y²可以表示为：

    (+ (square x)(square y) )

    然后我们利用这个构件去定义一个过程，给它两个实际参数，让他产生这两个数的平方和：

    (defined (sun-of-squares x y)
    (+ (square x ) (square y) ))
    ** //如果放在javascript中 **
    function sun-of-squares(x,y){
    return square(x)+square(y)
    }

    调用：

    (sum-of-squares 3 4) //25

    然后可以继续用它作为构件，去构造其他过程：

    (define (f a) )
    (sum-of-square (+ a 1 ) (* a 2) )

    调用：

    (f 5) //136

    相当于JavaScript中的：

    function f(a){
    return sum-of-square(a+1,a*2)
    }

  3. 复合过程的使用过程与基本过程（例如+和*）完全一样，如果但看sum-of-square很难分辨出究竟是直接做在解释器中还是一个复合过程。

1.1.5 ###### 过程应用的代换模型
  1. 解释器的工作方式：与以运算符名为基本过程的组合式一样的计算过程 - 将组合式的各个元素求值，然后将得到过程应用于实际参数。
  2. 对于复杂过程，过程应用的计算过程是：将复合过程引用于实际参数，就是再将过程体中的每个形参用相应的实参取代之后，对这一过程体求值。

书上的实例：

​ 下面组合式的求值：

​ (f 5)

​ 首先提取f为：

​ ( sum-of-squared ( + a 1) ( * a 2 ) )

​ 将实参5替换形参：

​ ( sum-of-squared ( + 5 1) ( * 5 2 ) )

​ 这个时候，问题被归约为对另一个组合式的求值，有两个运算对象，运算符是sum-of-squares，这里牵扯到2个问题：

​ 1.必须对其中的运算符求值，以便得到应该去应用里的哪个过程。

​ 2.需要求值两个运算对象，以得到过程的实际参数。

​ 将(+ 5 1 )和(* 5 2 )替换sun-of-squares中的形参：

​ +( ( square 6 ) (square 10) )

​ 通过square的定义( * x x )又可以归纳为：

​ ( + ( * 6 6 ) ( * 10 10 ) )

​ 然后相乘继续归纳为：

​ ( + 36 100 )

​ 最后得出结果:

​ 136

​ 这种描述的计算过程被称为过程应用的代换模型，强调两点：

​ 1. 代换的作用只是为了帮助领会过程调用中的情况，而不是对解释器实际工作方式的 具体描述。通常的解释器都不采用直接操作过程的正文，用值去代换形参的方式去完成 对过程调用的求值。

​ 2.这里的模型仅仅是个不完全模型。

应用序和正则序

​ 另一种求值模型：先不求出运算对象的值，直到实际需要他们的时候再去做，采用这种求值方式的话我们应该首先用运算对象表达式去代换形参，直到得到一个只包含基本运算符的表达式，然后再去执行求值。

​ 实例：比如( f 5 )应按照以下步骤逐步展开：

​ ( sum-of-squares ( + 5 1 ) (* 5 2) )

​ (+ ( square ( + 5 1 ) ) ( square( * 5 2 ) ))

​ (+ ( * ( + 5 1 ) ( + 5 1 ) ) (* ( * 5 2 )( * 5 2 ) ) )

​ 再进行运算：

​ (+ ( * 6 6 ) ( * 10 10 ) )

​ (+ 36 100 )

​ 136

​ 这种完全展开而后规约的求值模型叫做正则序求值

​ 与其相对的是解释器里实际使用的先求值参数而后应用的方式，他称为应用序求值

1.1.6 条件表达式和谓词

分情况分析结构：在Lisp中有一种针对分类情况分析的特殊形式，称为cond(条件)。

使用形式：

用JavaScript可表示为：

一般形式为：

首先包含符号cond,在它之后跟着一些称为子句的用括号括起来的表达式对偶(<p> <e>)

在每个对偶中的第一个表达式(<p>)是一个谓词(表达式)，它的值将被解释为true/false，第二个表达式(<e>)是序列表达式，这个值作为整个条件表达式的值。

求值方法：

1. 求值`<p1>`，如果为false
2. 求值`<p2>`，如果为false
3. 求值`<p3>`，.....
4. 结果为true，返回`<e>`的值 或者 无法找到为true的`<p>`，cond的值没有定义。

另一种写法：

简单来说就是，当x<0的时候返回-x，否则返回x，else是一个特殊符号，可以用在cond的最后一个子句中的<p>的位置，当该cond前面的所有子句都被跳过，就会返回最后子句中的<e>值。

换成JavaScript写法：

所有永远都求出真值的表达式都可以用在这个<p>的位置上

再一种写法：

这里使用特殊形式if，是条件表达式的一种受限形式，适用于分情况分析中只有两个情况的需要。

一般形式：

(if <predicate><consequent><alternative>)

解析形式：

​ 1.从求值<predicate>开始。

​ 2.如果<predicate>得到true。

​ 3.求值<consequent>并返回，否则求值<alternative>并返回。

除了基本谓词< = >之外，还有一些逻辑复合运算符，利用他们可以构造各种复合谓词，最常用的三个：

​ 1.(and <e1> ... <en>)-解释器将从左到右一个个求值<e>，如果某个<e>为false，则该表达式为false并break，如果所有<e>都为true，则该表达式的值就是最后的<e>。

​ 2.(or<e1> ... <en>)-解释器从左到右挨个求值，如果<e>为true，则表达式以该表达式作为值且break，如果所有<e>都为false则该表达式为false。

​ 3.(not <e>)-如果<e>求出值为false，则表达式为true，否则为flase

注：or和and都是特殊形式，因为字表达式未必求值，但是not则是普通的过程

实例：

x的值位于区间5<x<10的条件可写为：

(and ( > x 5 )( < x 10 ))
