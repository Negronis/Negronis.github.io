Redux

状态管理，所有State都应该被保存在一个单一对象中。

State

整个应用的数据源。

整个应用的 state 被储存在一棵 object tree 中，并且这个 object tree 只存在于唯一一个 store 中。

state是只读的

Store

负责把Action和Reducer联系在一起，一个Redux应用只应该有一个单一的store

Reducer

就是一个函数，根据Action来决定返回什么值，接受旧的state和Action，返回新的state

目前不要再Reducer中做：

* 修改传参
* 执行带有副作用的操作(请求/路由)
* 调用非纯函数(Date.now()/Math.random())
